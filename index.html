ما هو OSI Model؟

نموذج OSI (Open Systems Interconnection) هو إطار مفاهيمي يتكون من 7 طبقات يستخدم لفهم وتصميم بنية الشبكات. كل طبقة لها وظائف محددة وتتواصل مع الطبقات المجاورة.

Application Layer - طبقة التطبيقات

هي الطبقة الأقرب للمستخدم النهائي وتوفر واجهة بين التطبيقات وعمليات الشبكة.

الوظائف:

  • • توفير خدمات الشبكة للتطبيقات
  • • إدارة الاتصال بين التطبيقات
  • • التعامل مع البيانات للمستخدم

البروتوكولات:

  • • HTTP/HTTPS - تصفح الويب
  • • FTP - نقل الملفات
  • • SMTP - البريد الإلكتروني
  • • DNS - نظام أسماء النطاقات
6

Presentation Layer - طبقة العرض

مسؤولة عن ترجمة وتنسيق وتشفير البيانات لتكون مفهومة لطبقة التطبيقات.

الوظائف:

  • • تشفير وفك تشفير البيانات
  • • ضغط البيانات وفك الضغط
  • • ترجمة تنسيقات البيانات

أمثلة:

  • • SSL/TLS - التشفير الآمن
  • • JPEG, GIF - تنسيقات الصور
  • • MPEG - تنسيقات الفيديو
  • • ASCII, Unicode - تشفير النصوص
5

Session Layer - طبقة الجلسة

تدير الجلسات والحوارات بين الأجهزة، وتنظم بدء وإنهاء الاتصالات.

الوظائف:

  • • إنشاء وإدارة وإنهاء الجلسات
  • • المزامنة بين الأجهزة
  • • استعادة الجلسة عند الانقطاع

البروتوكولات:

  • • NetBIOS
  • • RPC - Remote Procedure Call
  • • PPTP - Point-to-Point Tunneling
4

Transport Layer - طبقة النقل

توفر نقل البيانات الموثوق من نقطة إلى أخرى، مع التحكم في تدفق البيانات واكتشاف الأخطاء.

الوظائف:

  • • تقسيم البيانات إلى Segments
  • • التحكم في تدفق البيانات
  • • اكتشاف وتصحيح الأخطاء
  • • ترقيم المنافذ (Port Numbers)

البروتوكولات:

  • • TCP - موثوق ومنظم
  • • UDP - سريع بدون ضمان
  • • SCTP - Stream Control
3

Network Layer - طبقة الشبكة

مسؤولة عن توجيه البيانات من المصدر إلى الوجهة عبر شبكات متعددة.

الوظائف:

  • • التوجيه (Routing) بين الشبكات
  • • العنونة المنطقية (IP Addressing)
  • • تحديد أفضل مسار للبيانات
  • • تجزئة الحزم (Packet Fragmentation)

البروتوكولات:

  • • IP - Internet Protocol (IPv4/IPv6)
  • • ICMP - رسائل التحكم
  • • ARP - Address Resolution
  • • Routing Protocols (OSPF, BGP)
2

Data Link Layer - طبقة ربط البيانات

توفر نقل البيانات الموثوق بين عقدتين متصلتين مباشرة في الشبكة المحلية.

الوظائف:

  • • التأطير (Framing) للبيانات
  • • العنونة الفيزيائية (MAC Address)
  • • اكتشاف الأخطاء والتحكم
  • • التحكم في الوصول للوسط

التقنيات:

  • • Ethernet
  • • Wi-Fi (802.11)
  • • PPP - Point-to-Point Protocol
  • • Switches تعمل في هذه الطبقة
1

Physical Layer - الطبقة الفيزيائية

الطبقة الأساسية التي تتعامل مع النقل الفعلي للبيانات عبر الوسائط المادية.

الوظائف:

  • • نقل البتات (Bits) عبر الوسط
  • • تحديد المواصفات الكهربائية
  • • تحديد نوع الكابلات والموصلات
  • • معدل الإرسال وتوقيت الإشارات

الأجهزة والوسائط:

  • • كابلات (UTP, Fiber Optic)
  • • Hubs, Repeaters
  • • موجات الراديو والإشارات
  • • Network Interface Cards (NIC)

Client-Server Model

نموذج العميل-الخادم هو بنية شبكة حيث يطلب العميل (Client) خدمة أو مورد من الخادم (Server)

كيف يعمل النموذج؟

1. Client (العميل)

المستخدم يفتح المتصفح ويكتب عنوان الموقع (URL)

2. Request (الطلب)

المتصفح يرسل HTTP Request للخادم عبر الإنترنت

3. Server (الخادم)

الخادم يستقبل الطلب ويعالج البيانات

4. Processing (المعالجة)

الخادم يبحث عن الملفات المطلوبة ويجهز الرد

5. Response (الرد)

الخادم يرسل HTTP Response يحتوي على البيانات المطلوبة

6. Rendering (العرض)

المتصفح يعرض الصفحة للمستخدم
